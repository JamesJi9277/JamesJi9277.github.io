Redis的一些总结

Redis 是一种常用的 Key-Value 数据库，因为比 Memcached 支持的数据结构更丰富，经常被用来存放热数据及其间的相互关系，比如日记 ID 与评论 ID 的一对多关系。

在最近的project中，我打算将 Redis 当作 ID 缓存来用的，从中可以方便快速的取到所需的实体 ID，然后再去 Memcached 或者 MySQL 中取详细数据。因为redis是基于缓存操作的，尤其是在去重等预处理过程中有十分明显的性能优势。

Redis 的常用数据结构有五种：String、Hash、List、Set 和 Sorted Set。一般数据集的特点不外乎有序还是无序，有索引还是无索引，唯一还是不唯一。Redis提供的常用数据类型恰好就满足了这些需求。

无序 + 无索引 + Value 唯一 = Set

无序 + Key 索引 + Key 唯一 = Hash

无序 + 位置索引 + 不唯一 = List

有序 + Key 和位置索引 + Key 唯一 = Sorted Set

在实际工程应用中，元素无序、唯一并且经常需要进行集合操作时选用 Set；一一对应的无序关系一般就选用 Hash 了，可以用 HMSET 和 HMGET 命令来一次设置或获取多个值；堆栈或者队列就选用 List 实现，可以选用 BLPOP 或者 BRPOP 命令实现阻塞型的出栈或出队列操作；有排序依据的数据集就选用 Sorted Set。
